# 需求文档

## 目标

Zen Story2Script 的目标是提供一套小说转结构化剧本 YAML 的 AI Agent 工作台。用户输入至少 3 章小说文本后，系统生成可编辑、可复制、可下载的剧本 YAML 草稿，并展示 Agent 步骤、质量报告和 warnings。

当前 MVP 面向：

- 小说作者：快速判断章节是否适合改编为短剧或影视剧本。
- 编剧助理：从长文本中提取人物、事件、场景和对白草稿。
- 短剧团队：把连载片段拆成可讨论的分场结构。
- 内容工作室：沉淀结构化改编素材，后续进入人工打磨。

## 核心用户流程

1. 用户注册或登录。
2. 用户填写小说标题。
3. 用户粘贴至少 3 章小说正文。
4. 用户选择改编目标：短剧、影视剧本或分场大纲。
5. 用户选择转换模式：快速模式或完整 ReAct。
6. 用户填写可选风格提示。
7. 用户开始转换。
8. 前端通过 SSE 展示 Agent 进度。
9. 用户查看 YAML、质量报告和 warnings。
10. 用户复制 YAML 或下载 `.yaml` 草稿。

## 功能需求

### 认证

- 支持注册、登录、登出和当前用户查询。
- 注册需要邀请码。
- 登录态通过 HTTP-only Cookie 保存。
- 转换接口必须要求登录。
- 本地 `dev` profile 可使用默认邀请码 `dev-invite` 演示。

### 小说输入

- 支持纯文本粘贴。
- 支持标题和风格提示。
- 支持中英文常见章节标题识别。
- 至少需要 3 章后才能开始转换。
- 前端应展示字数、章节数和是否满足最低输入要求。

支持示例：

- `第一章 归来`
- `第 1 章 归来`
- `# 第一章 归来`
- `Chapter 1 The Station`

### 改编目标

| 值 | 中文说明 |
| --- | --- |
| `short_drama` | 短剧 |
| `screenplay` | 影视剧本 |
| `scene_outline` | 分场大纲 |

### 转换模式

| 值 | 说明 |
| --- | --- |
| `fast` | 快速模式，适合联调、预览和快速草稿 |
| `react` | 完整 ReAct，执行更完整的章节解析、故事分析、场景规划、YAML 生成、校验和修复 |

### Agent 输出

输出必须包含：

- YAML 文本。
- Schema 版本。
- warnings。
- qualityReport。
- agentTrace。

YAML 必须符合 [剧本 YAML Schema v1.0](screenplay-yaml-schema.md)。

### 前端工作台

已登录后显示三段式工作台：

- 顶部 App Header：产品名、用户、语言切换、登出、连接状态。
- 左侧输入配置区：标题、正文、改编目标、风格提示、输入状态、开始按钮。
- 右侧结果进度区：模式选择、SSE 进度、Agent 步骤、YAML 预览、复制下载、质量报告、warnings。

未登录时显示认证入口：

- 登录 / 注册切换。
- 注册需要邀请码。
- 保留创作者工作室气质，不做营销落地页。

### 降级能力

- 后端 `dev` profile 可无密钥启动并返回 fallback 结果。
- 当前端无法连接后端时，`api.js` 保留 local mock fallback，便于前端演示。
- UI 必须明确显示本地 mock 或后端 dev fallback 状态。

## 非功能需求

- 本地可复现：README 能指导评审启动前后端。
- 功能稳定：登录、注册、登出、转换、SSE、复制、下载不能被 UI 改造破坏。
- 输出可编辑：YAML 不混入 Markdown 代码块标记。
- 安全边界清晰：不提交密钥、令牌、数据库密码、真实邀请码或未授权文本。
- 响应式布局：桌面左右工作台，移动端上下布局。
- 文案自然：中文优先，保留中英文切换。

## MVP 边界

当前已覆盖：

- 轻量账号体系。
- 登录保护转换接口。
- 小说转剧本 YAML。
- Agent 步骤展示。
- SSE 流式进度。
- YAML 复制和下载。
- 质量报告和 warnings。
- dev fallback 和 local mock fallback。
- 内置改编知识与可选 RAG。

当前不做：

- 团队空间。
- 作品历史保存。
- 多人协作。
- 可视化剧本编辑器。
- Word、PDF、Fountain 导出。
- 长篇全集批处理。
- 内容授权自动检测。

这些能力可以作为后续版本扩展。

## 验收标准

功能验收：

- 未登录访问转换接口返回 `401`。
- 注册或登录后可调用转换接口。
- 输入不足 3 章时前端阻止转换并提示。
- 输入示例小说后可生成结构化 YAML。
- SSE 能展示步骤并最终展示 YAML。
- YAML 可复制、可下载。
- warnings、qualityReport、agentTrace 可展示。

文档验收：

- README 能指导本地启动。
- examples 使用原创文本，不含敏感信息。
- Schema 文档与后端 v1.0 校验器一致。
- 本地开发文档说明 dev fallback、真实模型联调和敏感信息边界。

安全验收：

- 不提交 `application-local.yml`。
- 不提交真实 API Key、token、数据库密码、真实邀请码。
- 不提交真实用户上传内容或未授权版权文本。
